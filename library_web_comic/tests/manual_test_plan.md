# 手動テスト計画

## 目的
主要な機能（同期フォールバック、エクスポート/インポート、競合解決、仮想リスト）を手動で検証する。

## 前提
- Chrome（または Chromium）でデベロッパーモードを有効にし、拡張を読み込んでいること。
- popup.html を開けること。

---

## テストケース一覧

### 1. 基本操作
- 手順
  1. ポップアップを開く。
  2. タイトル・URL を入力して「本棚に追加」。
  3. 追加されたアイテムが一覧に表示されることを確認。
  4. 編集・移動・削除・リンクコピーを試す。
- 期待結果
  - 各操作が即時に反映され、`chrome.storage.local` に保存される。

### 2. タグ履歴と候補
- 手順
  1. 複数のタグを追加して履歴に登録。
  2. タグ入力で候補が表示されることを確認。
- 期待結果
  - 候補が表示され、選択でフォームに追加される。

### 3. ページメタ取得
- 手順
  1. 任意のサイトを開き、ポップアップの「ページ情報を取得」を押す。
- 期待結果
  - タイトル・画像・URL・サイト名がフォームに入力される（CSP 等で取得できない場合はエラーメッセージ）。

### 4. storage.sync 容量超過の再現
- 手順
  1. コンソールで `__simulateLargeSyncSave('test_big', 300)` を実行（開発用ユーティリティ）。
  2. `saveHybridWithFallback` を使う操作（タグ保存など）を行う。
- 期待結果
  - `quota` エラーが発生した場合、データは `storage.local` に退避し、トーストで通知される。`mangaErrorLogs` にログが残る。

### 5. チャンク化保存と復元
- 手順
  1. 大きなオブジェクトを `saveSyncChunks` で保存（開発用）。
  2. `loadSyncChunks` で復元を試みる。
- 期待結果
  - 正常に復元される。途中で失敗した場合は local に退避される。

### 6. エクスポート / インポート
- 手順
  1. 「エクスポート」で JSON をダウンロード。
  2. データを一部編集して「インポート」。
- 期待結果
  - インポート前に確認ダイアログが出る。インポート後、データが復元される。

### 7. インポート競合解決
- 手順
  1. 既存データを残した状態で、別のバックアップ JSON をインポートする（差分があるもの）。
  2. 競合ダイアログが表示されることを確認し、各選択（上書き・保持・マージ）を試す。
- 期待結果
  - 選択に応じてデータが上書き・保持・マージされる。配列は ID ベースでマージされる。

### 8. 仮想リストの動作確認
- 手順
  1. 大量（数百〜千件）のダミーアイテムを `storage.local` に投入（スクリプトで生成）。
  2. ポップアップを開き、スクロールの滑らかさと初期描画時間を確認。
- 期待結果
  - 初期描画が高速で、スクロール時に DOM ノード数が限定されていること（開発者ツールで確認）。

### 9. 権限フロー
- 手順
  1. 任意のサイトで「このサイトで自動取得を許可」を押す。
  2. 許可後、`grantedHostsList` に反映されることを確認。
  3. 取り消し操作を行う。
- 期待結果
  - 権限の付与・取り消しが正しく反映される。

---

## ログと不具合報告
- 障害ログは `mangaErrorLogs` に蓄積される。必要に応じてエクスポートして解析すること。
- 不具合は再現手順・期待結果・実際の挙動を記載して Issue を作成してください。
